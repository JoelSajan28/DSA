# API Setup – FastAPI + Uvicorn

## Overview

This API layer is built using **FastAPI** and served using **Uvicorn**.

* **Python** acts as the runtime (executes the code)
* **FastAPI** is the framework (defines routes and generates API documentation)
* **Uvicorn** is the server (handles HTTP requests and forwards them to the FastAPI app)

---

## Architecture

```text
Client (Browser / BOB)
        ↓
Uvicorn (Server)
        ↓
FastAPI (Framework)
        ↓
Application Code (Endpoints)
        ↓
Response
```

---

## Key Concepts

### Runtime

* Python executes the application code

### Framework (FastAPI)

* Defines API routes (e.g., `/person`, `/events`)
* Handles request/response logic
* Automatically generates OpenAPI schema
* Provides Swagger UI for testing (`/docs`)

### Server (Uvicorn)

* Listens for incoming HTTP requests
* Forwards requests to FastAPI
* Returns responses to the client

---

## Setup Instructions

### 1. Create Virtual Environment

```bash
python3 -m venv .venv
source .venv/bin/activate
```

---

### 2. Install Dependencies

```bash
pip install fastapi uvicorn
```

---

### 3. Run the Server

```bash
uvicorn app:app --reload
```

* `app` → filename (`app.py`)
* `app` → FastAPI instance (`app = FastAPI()`)

## API Testing (Swagger UI)

FastAPI provides built-in Swagger UI:

```
http://127.0.0.1:8000/docs
```

* Test endpoints directly from the browser
* View request/response formats
* Inspect API schema

---

## Important Notes

* FastAPI does **not include a server**, so Uvicorn is required
* Swagger UI is generated by FastAPI and served via Uvicorn
* Ensure all endpoints return valid JSON responses
